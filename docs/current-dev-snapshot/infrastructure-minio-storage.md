# Infrastructure MinIO Storage

## Environment

Keep storage credentials in `.env` or the runtime environment only:

```bash
APP_MINIO_ENDPOINT=http://localhost:9000
APP_MINIO_ACCESS_KEY=minioadmin
APP_MINIO_SECRET_KEY=minioadmin
APP_MINIO_SIGNING_REGION=minio
APP_MINIO_BUCKET=smartpark
```

Do not commit real access keys or secret keys.

## Local Docker Compose

`compose.yaml` includes a local `minio` service:

- API: `http://localhost:9000`
- Console UI: `http://localhost:9001`
- Data volume: `minio_data`
- Bucket: `smartpark`

The backend creates the bucket on startup when MinIO/S3 settings are configured. If MinIO is not configured, the app can still start, but storage endpoints reject presign operations.

## Object Key Convention

Manager uploads are scoped to the authenticated tenant:

```text
tenants/{tenantId}/{folder}/{uuid}-{safeFileName}
```

The frontend cannot provide arbitrary object keys for upload. Download presign requests must use an object key under the current tenant prefix.

## Presign Upload

`POST /manager/storage/presign-upload`

```json
{
  "fileName": "b1-map.png",
  "contentType": "image/png",
  "folder": "floor-maps"
}
```

Response:

```json
{
  "objectKey": "tenants/{tenantId}/floor-maps/{uuid}-b1-map.png",
  "uploadUrl": "http://localhost:9000/smartpark/...",
  "method": "PUT",
  "headers": {
    "Content-Type": "image/png"
  },
  "expiresInSeconds": 900,
  "publicUrl": null
}
```

Allowed image content types:

- `image/png`
- `image/jpeg`
- `image/webp`

The file extension must match the content type.

## Presign Download

`GET /manager/storage/presign-download?objectKey=tenants/{tenantId}/floor-maps/{uuid}-b1-map.png`

```json
{
  "downloadUrl": "http://localhost:9000/smartpark/...",
  "expiresInSeconds": 900
}
```

## Manual Test

```bash
docker compose up -d postgres redis minio
```

Create an upload URL:

```bash
curl -X POST "$BASE_URL/manager/storage/presign-upload" \
  -H "Authorization: Bearer $MANAGER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"fileName":"b1-map.png","contentType":"image/png","folder":"floor-maps"}'
```

Upload a small image using the returned URL:

```bash
curl -X PUT "$UPLOAD_URL" \
  -H "Content-Type: image/png" \
  --data-binary @b1-map.png
```

Create a download URL:

```bash
curl "$BASE_URL/manager/storage/presign-download?objectKey=$OBJECT_KEY" \
  -H "Authorization: Bearer $MANAGER_TOKEN"
```

## Security Notes

- S3/MinIO credentials are server-side only.
- Presigned URLs expire after 15 minutes.
- Upload keys are generated by the backend.
- Download keys must remain inside `tenants/{tenantId}/`.
- `publicUrl` is currently `null`; callers should use presigned download URLs for private objects.
