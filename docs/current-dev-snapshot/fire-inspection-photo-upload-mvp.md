# Fire Inspection Photo Upload MVP

## Endpoint

`POST /staff/fire-inspections/photos/presign-upload`

Requires `STAFF`. Tenant and staff identity come from the authenticated session and approved staff workContext; clients cannot provide `tenantId` or storage folder.

Request:

```json
{
  "fileName": "fe-b1-001-check.jpg",
  "contentType": "image/jpeg"
}
```

Response:

```json
{
  "uploadUrl": "https://storage.example/presigned-put-url",
  "objectKey": "tenants/{tenantId}/fire-inspections/{staffId}/{uuid}-fe-b1-001-check.jpg",
  "method": "PUT",
  "headers": {
    "Content-Type": "image/jpeg"
  },
  "expiresInSeconds": 900
}
```

## Rules

- Object keys are generated by the backend under `tenants/{tenantId}/fire-inspections/{staffId}/`.
- Inspection submit accepts `photoObjectKey` only under `tenants/{tenantId}/fire-inspections/`.
- Allowed content types are `image/jpeg`, `image/png`, and `image/webp`.
- File extensions must match content type: `.jpg`/`.jpeg`, `.png`, or `.webp`.
- Storage credentials stay server-side only.
- If MinIO/S3 is not configured, the app still starts and presign calls return `STORAGE_NOT_CONFIGURED`.

## Upload Flow

1. Staff requests `POST /staff/fire-inspections/photos/presign-upload`.
2. FE uploads the binary file with `PUT uploadUrl` and the returned headers.
3. FE submits `POST /staff/fire-inspections` with `photoObjectKey`.
4. Manager logs render `photoDisplayUrl` from `GET /manager/fire-inspections/logs`.

Submit example:

```json
{
  "fireExtinguisherId": "extinguisher-id",
  "result": "OK",
  "pressureOk": true,
  "sealOk": true,
  "locationOk": true,
  "expiryOk": true,
  "photoObjectKey": "tenants/{tenantId}/fire-inspections/{staffId}/{uuid}-fe-b1-001-check.jpg",
  "note": "Looks good",
  "nextInspectionAt": "2026-07-01T00:00:00"
}
```

## Manager Display

`GET /manager/fire-inspections/logs` returns:

- `photoUrl`: legacy URL input, if provided.
- `photoObjectKey`: private storage object key, if provided.
- `photoDisplayUrl`: presigned download URL for object-key photos, or legacy HTTP(S) `photoUrl`.
- `photoUrlExpiresInSeconds`: set for presigned URLs, `null` for legacy public URLs.

If storage is unavailable when listing logs, the API still returns the log row with `photoObjectKey`; `photoDisplayUrl` is `null`.

Inspection log filters are null-safe: omitted or empty `extinguisherId`, `parkingId`, `floorId`, `result`, `from`, and `to` filters behave like no filter. Date range filters are only applied when supplied.

## Legacy Behavior

`photoUrl` is still accepted for backward compatibility. New mobile flows should use presign upload plus `photoObjectKey`.

## Local And Production Notes

- Local MinIO is available through `docker compose up -d minio` with API port `9000`.
- Backend creates the configured bucket on startup when storage settings are present.
- Production must configure `APP_MINIO_ENDPOINT`, access key, secret key, signing region, and bucket.
